\section{Introduction}

\subsection{De la version 1 à la version 2}

Les versions 1.xx de \texttt{pas-algo} n'étant pas satisfaisantes et provoquant de multiples \og warnings \fg, j'ai décidé de faire une nouvelle version, bien plus conviviale.

\medskip

Pour les versions 2.xx, l'extension repose essentiellement sur \texttt{tcolorbox} qui, au m\^eme titre que \texttt{tikz}, repose sur \texttt{pgf}.

Ainsi, \texttt{pas-algo V2.xx} ne charge que le package \texttt{tcolorbox}, avec les librairies \texttt{skins}, \texttt{theorems} et \texttt{breakable}.

\subsection{Installation}

Je vous conseille fortement de mettre à jour \texttt{pgf} car \texttt{tcolorbox} ne fonctionne pas correctement avec une version trop ancienne.

Concernant l'installation de \texttt{pas-algo V2.xx}, je vous conseille de l'installer dans un répertoire personnel suivant la structure conventionnelle \LaTeX.

\medskip

Sous Ubuntu, on pourra donc décompresser \texttt{pas-algoV2.zip} dans le répertoire :

\begin{verbatim}
./texlive/texmf-local/tex/latex/
\end{verbatim}

de sorte à avoir :

\begin{verbatim}
./texlive/texmf-local/tex/latex/pas-algo/pas-algo.sty
\end{verbatim}
\begin{verbatim}
./texlive/texmf-local/tex/latex/pas-algo/algo.png
\end{verbatim}
\begin{verbatim}
./texlive/texmf-local/doc/latex/pas-algo/pas-algo.pdf
\end{verbatim}
\begin{verbatim}
./texlive/texmf-local/doc/latex/pas-algo/pas-algo.tex
\end{verbatim}
\begin{verbatim}
./texlive/texmf-local/doc/latex/pas-algo/doc.codes.tex
\end{verbatim}
\begin{verbatim}
./texlive/texmf-local/doc/latex/pas-algo/doc.styles.tex
\end{verbatim}

Après installation, n'oubliez pas de taper la commande \texttt{texhash} dans le terminal pour mettre à jour la base de données des extensions.

\bigskip 

Avec Miktex (sous Windows) ou macTex (sous Mac OS), j'imagine que l'arborescence ressemble à ce qui est écrit précédemment.

\subsection{Paramètre général}

Vous allez le constater dans les portions de codes, une variable \texttt{\textbackslash algoDecal} intervient pour décaler le corps de chaque environnement.

Cette variable est une longueur, et peut \^etre changée comme ceci :

\begin{tcblisting}{codeTEX,title={Changement de la longueur algoDecal}}
\setlength\algoDecal{-1em} % valeur par défaut
\end{tcblisting}

En mettant une valeur supérieure à celle par défaut (donc à $-1$em), le décalage sera plus vers la droite.

En mettant une valeur inférieure à $-1$em, le décalage se fera plus vers la gauche.

\subsection{Remarque importante}

J'ai tenté de conserver au maximum la m\^eme syntaxe pour les commandes, mais  il m'a tout de m\^eme fallu changer pas mal de choses. Ainsi, il faudra effectuer quelques modifications à vos documents qui utilisaient la version 1.

J'en suis réellement désolé.


\section{L'environnement principal \og algo \fg{} }

\subsection{La structure générale}

\begin{tcblisting}{listing,title={L'environnement algo}}
\begin{algo}[label]{Titre de l'algorithme}
Ici, l'algorithme \ref{algo:label}.
\end{algo}
\end{tcblisting}

\begin{tcblisting}{listing,title=Un algorithme sans titre}
\begin{algo}[sstitre]{}
Ici, l'algorithme \ref{algo:sstitre} n'a pas de titre.
\end{algo}
\end{tcblisting}

\begin{tcblisting}{listing,title=Un algorithme non numéroté}
\begin{algo*}{}
Ici, l'algorithme n'est pas numéroté.
\end{algo*}
\end{tcblisting}

\enlargethispage*{3em}

L'option \texttt{label} est facultative ; elle sert à faire référence à l'algorithme à l'aide de la commande \textbackslash\texttt{ref\{algo:<nom du label>\}}.

\subsection{Changer le style de l'environnement}

Comme on peut le voir dans le code, le style de l'environnement \texttt{algo} est défini à l'aide de la macro \textbackslash\texttt{tcbset} (du package \texttt{tcolorbox}).

\end{tcblisting}

\section{Exemples}

\subsection{Exemple 1 : algorithme complet}

\begin{tcblisting}{listing,title=Somme de termes d'une suite arithmétique}
\begin{algo}[somsuitar]{Calcul d'une somme}
\begin{algoentries}
r est un nombre réel\\
u est un nombre réel\\
n est un entier naturel\\
i est un entier naturel\\
S est un nombre réel
\end{algoentries}
\begin{algoinit}
Affecter à S la valeur u\\
Entrer la valeur de r (raison de la suite arithmétique)\\
Entrer la valeur de u (premier terme de la somme)\\
Entrer la valeur de n (nombre de termes dans la somme)
\end{algoinit}
\begin{algobody}
\begin{algofor}{i}{1}{n-1}
Affecter à S la valeur S+(u+i*r)
\end{algofor}
\end{algobody}
\begin{algoend}
Afficher S
\end{algoend}
\end{algo}
L'algorithme \ref{algo:somsuitar} permet de calculer la somme  $u_p+u_{p+1}+u_{p+2}+\cdots+u_{p+n-1}$, où $(u)$ est une suite arithmétique de raison $r$. La valeur de $u$ saisie lors de l'initialisation est la valeur de $u_p$.
\end{tcblisting}
